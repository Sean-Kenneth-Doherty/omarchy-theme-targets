# Theme Target Convention

A target is a small adapter that turns the active Omarchy palette into one safe
app artifact.

Each target has:

- `manifest.toml`: target identity, safety class, output path, and template.
- `templates/*`: generated content with semantic palette placeholders.
- Apply behavior: render template and write only if the destination is missing
  or already has the generated header.
- Status behavior: report whether the generated file is present, missing,
  manual, or unsafe.
- Doctor behavior: validate color input, target output safety, and ownership.
- Rollback behavior: remove only generated files with the header.

Manifest shape:

```toml
id = "gtk"
name = "GTK artifact"
safety_class = "ui-theme"
description = "Safe GTK CSS artifact. It is not linked into gtk.css by default."
output = "{artifact_root}/gtk/gtk.css"
template = "gtk.css"
```

Supported output variables:

- `{home}`: current user home directory.
- `{artifact_root}`: `~/.config/omarchy-theme-targets`.

Templates use Python format placeholders. Semantic palette keys replace dots
with underscores, for example `{surface_base}`, `{accent_primary}`, and
`{focus_ring}`. Raw Omarchy colors such as `{color0}` and `{foreground}` are
also available.

Every generated file starts with:

```text
# Generated by omarchy-theme-targets.
```

That header is the contract that makes apply and rollback reversible.
